# 数据源扩展可行性调研实施报告

> **调研日期**：2026-07-21
> **调研方法**：目标平台实际访问 + Playwright 浏览器抓包 + 接口实测调用 + 公开资料交叉验证
> **调研范围**：全国公共资源交易平台、20+ 省级公共资源交易平台、央企/国企采购平台、前置线索源
> **业务前提**：所有候选数据源以"有利于管道销售"为筛选标准（管材品类：PVC/HDPE/PPR/PE/球墨铸铁管/钢管/波纹管等；目标场景：市政给排水、燃气、污水、水利、电力通信保护管等）
> **关联文档**：`docs/数据源调研总结.md`、`docs/未实现数据源-可行性调研报告.md`

## 一、调研背景、目标与方法

### 1.1 背景

项目已实现 14 个数据源，但存在三个明显边界：

1. **地域边界**：公共资源交易类仅覆盖广东、浙江、四川、江苏 4 省，全国尚有 27 个省级区域未覆盖；
2. **时效边界**：现有最早的前置信号为"项目审批/备案"（fdtz）与"环评公示"（eia），尚未利用"招标计划"这一法定提前 30 天公开的信息类型；
3. **行业边界**：央企/国企自有采购平台（燃气集团、水务集团、电网、运营商）尚未触及，而这些主体是管材的直接采购方。

### 1.2 调研目标

- 评估"各省公共资源交易平台"批量扩展的可行性（用户明确提出）；
- 发掘其他有利于管道销售的资讯类型并验证可行性；
- 对每个候选源给出明确结论：可行（含接口规格）/ 待抓包（含证据）/ 不可行（含原因）；
- 形成可落地的分级实施路线图。

### 1.3 验证等级定义

本报告所有结论按证据强度分三级，严禁混用：

| 等级 | 含义 | 采信方式 |
|------|------|----------|
| **已实测** | 通过浏览器抓包/接口实际调用获得请求参数与响应结构 | 可直接进入开发 |
| **高置信度** | 页面含明确的厂商/架构特征（如新点组件路径、分类编码风格），但接口未实测 | 需 0.5-1 天抓包确认后开发 |
| **方向性** | 仅有公开资料/搜索摘要支撑，未做任何技术验证 | 立项前必须补做验证 |

### 1.4 调研方法

1. 目标平台首页/列表页/搜索页实际访问，识别技术架构（SSR / SPA / 厂商特征）；
2. Playwright 浏览器抓取真实 XHR 请求，获取接口地址与参数格式；
3. 在页面上下文内重放接口调用，验证关键词搜索、省份过滤、分页、详情完整性；
4. 裸 HTTP 访问测试，评估 WAF/反爬强度；
5. 与项目现有 14 个数据源对比，评估重叠度与增量价值。

## 二、现有采集体系盘点与缺口分析

### 2.1 已实现数据源（14 个）

| 类别 | 数据源 | 覆盖 | 线索时效位置 |
|------|--------|------|--------------|
| 政府采购 | ccgp 中国政府采购网、gdgpo 广东政采网 | 全国 / 广东 | 采购公告发布时 |
| 公共资源交易 | ggzyjy 广东（粤公平）、ggzyjy_zj 浙江、ggzyjy_sc 四川、ggzyjy_js 江苏 | 4 省 | 招标公告发布时 |
| 前置审批 | eia 环评公示（广东多城市）、fdtz 广东发改审批 | 广东 | 立项/环评阶段（最早） |
| 住建系统 | gzfcj 广州施工许可/竣工验收、gdcic 广东住建厅 | 广东 | 施工阶段 |
| 行业垂直 | pipebiz 管道商务网、bjx 北极星环保网 | 全国 | 招标公告发布时 |
| 客户/画像 | poi 高德 POI（经销商）、qcc 企查查（企业画像） | 广东 / 全国 | 非项目线索 |

### 2.2 缺口分析（三维度）

**维度一：地域缺口（最突出）**

- 全国 31 个省级区域 + 新疆兵团，已覆盖 4 省（粤浙苏川），覆盖率约 13%；
- 未覆盖区域中，湖北、河南、广西、安徽、江西、贵州、云南、重庆、福建等均为市政建设活跃省份；
- 山东曾评估为不可行（旧平台 502、新系统需登录），维持放弃结论。

**维度二：时效缺口**

- 现有线索集中在"招标公告发布"时点，此时距开标通常仅 20 天左右，销售介入窗口紧张；
- 发改委《招标投标领域公平竞争审查规则》及各省落实文件要求：依法必须招标的项目应在招标公告发布前至少 30 日发布**招标计划**——这是法定的、比招标公告更早的公开信号，且各省公共资源平台均设有"招标计划"栏目，目前完全未采集。

**维度三：行业缺口**

- 燃气集团（华润燃气、中国燃气、港华等）、水务集团（北控水务、首创环保等）、电网（国网/南网）、运营商（移动/电信/联通）是燃气管、给水管、电力保护管、通信管的**直接采购方**；
- 这些主体的大型集采部分经公共资源平台进行（可被全国平台覆盖），但年度框架集采、小额采购多在自有电子采购平台发布，构成独立增量。

### 2.3 缺口结论

扩展空间显著，且存在"一个源覆盖全国"的杠杆型机会（见第三章）。按优先级排序的扩展路径：全国聚合平台 → 招标计划（零成本）→ Epoint 同构省份低成本复制 → 重点省份单独适配 → 行业垂直平台。

### 3.2 核心接口规格（已实测）

**列表/搜索接口**：

```
POST https://www.ggzy.gov.cn/information/pubTradingInfo/getTradList
Content-Type: application/x-www-form-urlencoded; charset=UTF-8
```

| 参数 | 实测值/说明 |
|------|------------|
| `SOURCE_TYPE` | `1`=交易公告；`2`=成交公示 |
| `DEAL_TIME` | 时间快捷档位（`01`=当天、`02`=近三天、`06`=近一月等），配合自定义区间使用 |
| `TIMEBEGIN` / `TIMEEND` | 自定义时间区间，格式 `YYYY-MM-DD`，实测生效 |
| `PAGENUMBER` | 页码，从 1 开始 |
| `FINDTXT` | 搜索关键词，实测"管道"（URL 编码）生效 |
| `DEAL_PROVINCE` | 省份过滤，6 位行政区划代码（如 `440000`=广东），实测过滤生效 |

### 3.3 详情页结构（已实测）

详情页为静态 HTML，分两层：

| 路径 | 内容 | 实测结论 |
|------|------|----------|
| `/information/deal/html/a/{省份码}/{信息类型}/{日期}/{id}.html` | 元信息页：标题、项目编号、信息来源省、附件清单（招标文件 .BZBJ 等） | 无正文 |
| `/information/deal/html/b/{同上}.html` | **完整公告正文** | 字段极完整 |

**/b/ 正文实测字段**（以昆明燃气管道改造项目为例）：

- 招标项目名称、资金来源（国资 100%）、建设规模；
- 公共资源交易行业分类（市政工程）、工程类型（施工）、招标方式（公开招标）；
- **建设单位**：云南中石油昆仑燃气有限公司；**经办人**：王平；**办公电话**：18687155319；
- 招标代理机构：中国石油物资有限公司；项目负责人：郑刚；经办人：薛文康；办公电话：18881498505；
- 标段编号、**标段合同估算价 731.49 万元**、计划工期 180 日历天；
- 资质要求：市政公用工程施工总承包二级及以上 + **压力管道安装 GB1 级**；
- 开标时间/地点、招标文件获取截止时间。

对管道销售的价值：业主名称、联系人、电话、预算、工期、开标时间一次集齐，且"压力管道 GB1""输水管""燃气管网"等关键词直接标识管道需求场景。

### 3.4 反爬评估与采集方案

| 测试 | 结果 |
|------|------|
| 裸 HTTP 直接请求（非浏览器 TLS 指纹） | **TCP 连接被强制关闭**（WAF 拦截，首页/接口均如此） |
| Playwright Chromium 页面内 fetch 重放 | 正常返回 JSON |
| 频率限制 | 未实测到触发阈值，建议沿用项目标准随机延迟 |

> **2026-07-22 实施实测更新**：开发完成后真实联调发现，WAF 拦截具波动性——当日裸 httpx（带浏览器 UA + Referer + Origin）POST `getTradList` 可直接返回 `{"code":200,...}`，首页 GET 亦 200，且会话页未下发任何 Cookie。故 `scraper/ggzy_national/` 采用**懒回退**设计：默认直接 httpx 调用，不预启动 Playwright；仅当请求被 WAF 拦截（HTTP 403/493 或 `ConnectError`）时由 `api.py` 自动调用 `refresh_cookies()` 重建 Playwright 会话并带 Cookie 重试。上表"裸 HTTP 被强制关闭"为调研当日观察，保留作历史记录；采集器对两种状态都兼容。

**推荐采集方案**（复用项目成熟模式，参照 `scraper/pipebiz/browser.py`）：

1. Playwright 启动 Chromium 访问 `https://www.ggzy.gov.cn/deal/dealList.html`，建立会话并取得 Cookie；
2. httpx 携带 Cookie + 浏览器 UA，按关键词 × 时间区间循环调用 `getTradList`（默认全国拉取，也可按省份过滤做定向补充）；
3. 对每条记录请求 `/b/` 正文页解析详情；
4. Cookie 失效（响应非 JSON / 连接被重置）时自动回退 Playwright 重建会话。

### 3.5 与现有源的重叠与去重

- 全国平台数据来自各省平台同步，与 ccgp、gdgpo、粤公平、浙苏川 Epoint 存在**必然重叠**；
- 重叠不是缺陷而是兜底：已覆盖 4 省的增量有限，但未覆盖 27 省全部经此源接入；
- 项目已有三层去重机制（URL / 标题 / 内容哈希），跨省同项目标题一致时可自然去重；建议在采集器内将 `province` 写入 `Lead.region`，并保留 `id`（32 位 hash）作为 URL 级去重键。

### 3.6 土地线索附加价值

列表页业务类型含"土地使用权"（出让公示、成交宗地），数据源含自然资源部。**土地成交 = 新地块 = 未来 6-18 个月的新建项目**，是最早的可建设信号。可配置业务类型参数一并采集，替代 landchina.com（详见第七章 7.3）。

### 3.7 结论与工时

**可行，优先级 P0 —— 已实现（2026-07-22）**。一个源覆盖全国 31 省 + 央企/部委数据，接口 JSON 化、字段完整、支持关键词/省份/时间三维过滤。已完成 `scraper/ggzy_national/`（api/parser/detail/regions/utils + registry/config 注册），真实联调验证：单关键词 1 页采得 20 条跨 8 省线索，含业主/联系人/电话/预算/开标时间，附件 PDF 成功下载，二次运行去重 0 新增。复用 base/lifecycle/三层去重基础设施。

## 四、P1：招标计划采集（现有多源零成本扩展）—— 已实测存在性

### 4.1 政策依据与业务价值

依法必须招标的项目应在招标公告发布前至少 30 日发布招标计划（国家发改委相关规定及各省落实文件）。对销售而言：

- **最早的项目信号**：比招标公告早约 1-4 周，销售可提前联系业主、跟踪项目、准备样品与资质；
- **信息要素够用**：招标计划通常含项目名称、建设单位、估算金额、计划招标时间、联系人/电话；
- **竞争烈度低**：多数销售人员只在招标公告阶段介入，招标计划阶段介入者少。

### 4.2 各平台招标计划入口（本次实测/确认）

| 平台 | 招标计划栏目实测情况 | 等级 |
|------|---------------------|------|
| 湖北 hbggzy.cn | 首页设有"招标计划"独立栏目，实测可见当日多条（如"枝江市紧密型县域医共体强基工程建设项目招标计划"） | 已实测 |
| 湖北 hbggzyfwpt.cn | 有"招标计划""招标/资审文件提前公示"栏目 | 已实测 |
| 贵州 ggzy.guizhou.gov.cn | 首页"工程建设"板块含"招标计划"子栏目 | 已实测 |
| 北京 ggzyfw.beijing.gov.cn | 工程建设板块含"招标计划"栏目（搜索摘要证实） | 高置信度 |
| 全国平台 ggzy.gov.cn | 信息类型含招标计划类（`informationType` 体系内） | 高置信度 |
| 广东粤公平 / 浙苏川 Epoint | Epoint 分类编码体系中招标计划为独立分类（如浙江 `002001` 工程建设下子类），需在对应省份分类树中确认编码 | 高置信度 |

### 4.3 实施方式（增量改造，几乎零成本）

1. **Epoint 三兄弟（浙/苏/川）**：现有采集器 `CATEGORY_NUM` 为招标公告分类（如浙江 `002001001`）。新增招标计划分类编码作为第二组采集配置（常量级改动），复用同一套搜索/解析/去重逻辑；
2. **广东粤公平**：`scraper/ggzyjy/` 频道浏览模式增加招标计划频道参数；
3. **新源 ggzy_national（全国平台）**：`SOURCE_TYPE`/`informationType` 参数中包含招标计划类型，一并纳入；
4. **Lead 标记**：建议在线索标题前缀或备注字段标记"招标计划"，便于销售区分介入时点。

### 4.4 结论与工时

**可行，优先级 P1**。不改变现有架构，仅增分类配置与标记逻辑。预估工时 **1 天**（含四源分类编码确认与验证）。

1. 确认 ES 搜索 API 端点是否为 `/inteligentsearch/rest/esinteligentsearch/getFullTextDataNew`（或其变体）；
2. 确认请求格式属于哪一代（`wd/pn/rn` 旧格式 vs `keyword/pageNo/pageSize` 新格式——浙江已出现新格式升级，见 `scraper/ggzyjy_zj/__init__.py`）；
3. 确认分类编码前缀（工程建设招标公告、招标计划）与地区码表；
4. 确认 HTTPS 证书有效性（参照江苏 `verify=False` 的先例）；
5. 确认是否存在独立详情页解析差异。

### 5.4 实施方式

复用 `scraper/epoint/` 适配器架构：新增 `scraper/ggzyjy_hb/`、`scraper/ggzyjy_hn/`、`scraper/ggzyjy_gx/` 三个轻量子类（每省仅定义 `base_url`、`CATEGORY_NUM`、`TIME_FIELD`、`REGIONS`、SSL 策略，参照 `scraper/ggzyjy_zj/__init__.py` 仅 49 行的模式）。

### 5.5 结论与工时

**可行（抓包确认后），优先级 P1**。每省预估 **1-2 天**（含抓包 0.5 天 + 适配 0.5-1 天 + 验证 0.5 天），三省合计 **3-5 天**。

### 9.1 优先级排序

| 排名 | 数据源 | 优先级 | 验证等级 | 增量价值 | 预估工时 | 关键技术 | 需 Playwright |

| 2 | 招标计划采集（4 源增量） | **P1** | 已实测存在性 | 高（提前 30 天介入信号） | 1 天 | 分类配置 + 标记 | 否 |
| 3 | Epoint 同构省份（湖北/河南/广西） | **P1** | 高置信度 | 高（3 省 30+ 地市） | 3-5 天 | httpx（Epoint 框架） | 否 |
| 4 | 安徽公共资源交易（ggzy.ah.gov.cn） | **P2** | 已实测可访问 | 中（或由全国平台兜底） | 2-3 天 | httpx SSR 抓包 | 否 |

### 9.2 依赖与复用

- **全国平台先行**：其结果直接决定各省是否仍需单独开发——若全国平台对某省字段/时效达标，则该省单点采集优先级下调（§3.5 兜底逻辑）。建议作为第一阶段唯一硬性交付。
- **Epoint 三省**：依赖 `scraper/epoint/` 通用框架（base/search/parser/detail/utils），每省仅定义 `base_url`/`CATEGORY_NUM`/`TIME_FIELD`/`REGIONS`/SSL 策略（§5.4，参照 `scraper/ggzyjy_zj/__init__.py` 约 49 行模式）。
- **招标计划**：依赖各源分类编码确认（§4.3），与对应省份采集器同批验证即可。
- **安徽**：非 Epoint，需独立 SSR 适配器（参照 `scraper/ccgp/` 列表解析模式）；或直接放弃单采由全国平台覆盖。

### 9.3 推荐时间线

| 阶段 | 内容 | 预估总工时 | 产出 |
|------|------|:----------:|------|
| **第一阶段** | 全国平台（P0）+ 招标计划（P1） | 3-4 天 | 1 个杠杆新源 + 4 源招标计划增量，覆盖率 13% → 100% |
| **第二阶段** | Epoint 湖北/河南/广西（P1） | 3-5 天 | 3 省独立源（备选，与全国平台重叠由去重处理） |
| **第三阶段** | 安徽（P2） | 2-3 天 | 安徽独立源（备选，可由全国平台兜底） |
| **按需** | 央企/国企采购平台 | 逐家评估 | 自有平台小额采购增量 |

> 注：因全国平台可兜底覆盖所有省份，第二、三阶段各省单采为"精度/时效增强"性质，可按实际增量价值取舍，非必做。

### 9.4 风险

| 风险 | 说明 | 缓解 |
|------|------|------|
| 全国平台 WAF 持续性 | 裸 HTTP 被 TCP 强制关闭，依赖 Playwright 会话；Cookie 失效需回退重建 | 沿用 `scraper/pipebiz/browser.py` 模式，响应非 JSON/连接重置时自动回退（§3.4） |
| Epoint API 升级漂移 | 浙江/江苏已升级为 `keyword/pageNo/pageSize`，base `wd/pn/rn` 对新端点返回 `total=0`（`scraper/ggzyjy_zj/__init__.py:22-24` 已记录）；三省新源格式归属未定 | 每省抓包确认格式（§5.3 清单），按需覆盖 `_scrape_page` |
| 跨源数据重叠 | 全国平台与 ccgp/gdgpo/粤公平/浙苏川 Epoint 必然重叠 | 三层去重（URL/标题/内容哈希）兜底，`province` 写入 `Lead.region`，`id` 作 URL 级去重键（§3.5） |
| 土地/央企国企后续需求 | 若后续确需深度采集土地明细或某家央企全集采 | 重新立项评估，本报告结论为"由全国平台兜底/人工"，不阻塞当前路线 |

## 十、总结

本次扩展调研以"有利于管道销售"为标尺，对全国聚合平台、省级同构平台、行业垂直平台、央企国企采购平台、土地线索逐类给出证据分级结论：

- **一个 P0 杠杆源**：全国公共资源交易平台已实测可行，一源覆盖 31 省 + 央企/部委，2-3 天即可上线，是覆盖率从 13% 跃升的关键；
- **一个零成本 P1**：招标计划为法定提前 30 天信号，仅靠现有 4 源分类配置即可获得，1 天工时；
- **一个低成本 P1**：Epoint 同构的湖北/河南/广西可复用现有框架，3-5 天复制 3 省；
- **一个 P2 备选**：安徽自建系统可独立适配，亦可由全国平台兜底；
- **明确不推荐**：行业垂直（pipebiz/bjx 已实现、h2o 放弃、chinabidding 不推荐）、央企国企（方向性，不批量立项）、landchina（由全国平台覆盖）——避免无效投入。

由此形成可落地的分级实施路线图，回应 §1.2 调研目标第 4 条。推荐执行顺序：全国平台（P0）→ 招标计划（P1）→ Epoint 三省（P1，按需）→ 安徽（P2，按需）。
