### 3.2 浙江省公共资源交易平台（P1）

| 项目 | 信息 |
|------|------|
| 域名 | ggzy.zj.gov.cn |
| 访问验证 | 成功，HTTP/HTTPS 均正常 |
| 技术架构 | SSR 列表页 + EpointWebBuilder CMS 的 Elasticsearch 搜索 API |
| 反爬措施 | **无** — 无需 Cookie、无验证码、无特殊请求头 |

**API 端点**：

```
POST /inteligentsearch/rest/esinteligentsearch/getFullTextDataNew
Content-Type: application/json
```

**请求参数**：

| 参数 | 说明 |
|------|------|
| `pn` | 起始行号（分页偏移） |
| `rn` | 每页条数 |
| `sort` | 排序方式 |
| `condition[]` | 过滤条件数组 |
| `time[]` | 时间范围数组 |
| `wd` | 搜索关键词 |
| `cnum` | 分类号 |
| `isBusiness` | 固定值 `"1"` |

**响应结构**：JSON 格式

```json
{
  "result": {
    "totalcount": 575,
    "records": [
      {
        "title": "公告标题",
        "webdate": "2026-07-15",
        "infod": "杭州市",
        "infoc": "330101",
        "infoa": "行业码",
        "linkurl": "/jyxxgk/...",
        "categorynum": "002001001",
        "categoryname": "招标公告",
        "content": "公告摘要"
      }
    ]
  }
}
```

**详情页**：SSR HTML，含结构化表格（项目登记信息、招标人、预算等），URL 格式：`/jyxxgk/002001/002001001/{YYYYMMDD}/{UUID}.html`

**地区代码前缀**：`33xxxx`（11 地市：杭州=330101、宁波=330201 等）

**分类体系**：`catenum=002001`（工程建设），`mycatenum=002001001`（招标公告）

**数据量验证**：工程建设招标公告 2026-07-01 ~ 07-18：**575 条**

**推荐方案**：httpx 直连，不需要 Playwright

**预估工时**：2 天

## 五、三省 EpointWebBuilder 统一适配器架构

浙江、江苏、四川三省共享同一 EpointWebBuilder 平台，API 接口高度一致，建议采用统一适配器模式：

### 5.1 目录结构

```
scraper/epoint/
├── __init__.py
├── base.py          # 封装 getFullTextDataNew API 调用、参数构建、分页逻辑
├── adapters/
│   ├── __init__.py
│   ├── zhejiang.py  # 浙江配置（categorynum=002xxx, time field=webdate）
│   ├── jiangsu.py   # 江苏配置（categorynum=003xxx, time field=infodatepx, HTTP）
│   └── sichuan.py   # 四川配置（categorynum=002xxx, 21市州地区码）
└── parser.py        # 统一的 SSR HTML 详情页解析
```

### 5.2 适配器设计要点

- `base.py` 封装通用逻辑：API 请求构建、分页迭代、JSON 响应解析、地区码映射
- 每个 adapter 仅需配置（约 50-100 行）：
  - `BASE_URL`（域名）
  - `CATEGORY_PREFIX`（分类编码前缀，浙江/四川 `002`，江苏 `003`）
  - `TIME_FIELD`（时间字段名，浙江/四川 `webdate`，江苏 `infodatepx`）
  - `REGION_CODES`（地区代码映射表）
  - `HTTPS_VERIFY`（是否验证 SSL，江苏需 `False`）
- `parser.py` 统一处理详情页 HTML 解析（表格提取、字段映射）

### 5.3 与现有代码的复用关系

| 模块 | 复用程度 | 说明 |
|------|:--------:|------|
| 模块架构 | 可复用 | 遵循 `scraper/` 下各数据源的标准目录结构 |
| lifecycle 调度 | 可复用 | 接入 `scraper/lifecycle.py` 统一调度 |
| 去重逻辑 | 可复用 | 复用三层去重机制（URL/标题/内容哈希） |
| API 调用层 | **需重写** | EpointWebBuilder API 与广东 ggzyjy 不同 |
| 解析层 | **需重写** | 详情页 HTML 结构不同 |

与广东 ggzyjy（`scraper/ggzyjy/`）的代码可复用程度约 **40-50%**。

### 5.4 扩展性

新增省份只需增加一个 adapter 配置文件，核心逻辑完全复用。后续可覆盖的 EpointWebBuilder 省份包括：安徽、福建、广西、贵州、河南等（需逐一验证 API 可用性）。

## 七、实施优先级路线图

### 7.1 优先级排序

| 排名 | 数据源 | 优先级 | 可行性 | 增量价值 | 预估工时 | 关键技术 | 需 Playwright |

| 2 | 浙江 ggzyjy | P1 | 高 | 高（11 地市） | 2 天 | httpx REST API | 否 |
| 3 | 北极星环保网 | P1 | 中 | 中高 | 3-5 天 | Playwright WAF 绕过 | 是 |

- **浙江 → 四川 → 江苏**：建议按此顺序实现三省适配器。浙江作为首个 EpointWebBuilder 省份，完成 base.py 框架后，四川和江苏只需增加 adapter 配置文件

- **北极星环保网**：独立实现，可复用 pipebiz/fdtz 的 Playwright Cookie 方案
- **企查查 API**：独立实现，需企业实名认证和付费开通前置

| 阶段 | 内容 | 预估总工时 | 产出 |
