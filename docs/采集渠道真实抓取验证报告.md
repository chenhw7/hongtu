## 二、已修复问题

### 2.1 浙江/江苏 ES API 格式升级

**问题**：浙江和江苏的 `getFullTextDataNew` ES API 已升级为 `keyword/pageNo/pageSize` 参数格式，不再接受 epoint 基类构造的 `wd/pn/rn/cnum/condition/time` 格式（后者返回 total=0）。

**修复**：
- `scraper/ggzyjy_zj/__init__.py`：覆盖 `_scrape_page`，使用 `keyword/pageNo/pageSize` 格式构建请求体
- `scraper/ggzyjy_js/__init__.py`：同上
- `scraper/epoint/search.py`：给 `_post_search` 加 `api_url_override` 参数，支持子类自定义 API URL
- `scraper/epoint/parser.py`：日期字段从硬编码 `webdate` 改为动态读取 `scraper.TIME_FIELD`（江苏用 `infodatepx`）

**验证**：修复后浙江/江苏烟雾测试均 PASS。

## 三、未修复问题

### 3.1 pipebiz（中国管道商务网）

**失败原因**：Playwright 成功启动并过了 JS Challenge，但页面内容存在编码问题。

**详细诊断**：
- Playwright 返回 HTML 长度 20348 字符，有 60 个 `<li>` 标签
- 页面 charset 声明为 utf-8，但实际内容为乱码（GB2312/GBK 编码被按 UTF-8 解码）
- 搜索 URL 模板加载失败（页面仍在导航和变更内容）
- parser 的 3 个策略（列表容器→表格行→fallback链接）都返回空列表
- 页面日期模式为 0（乱码文本中无法识别 YYYY-MM-DD）

**修复方向**：
1. 调查 Playwright 页面编码问题（可能需要指定 encoding 或等待页面稳定后再取 content）
2. 更新 parser 选择器适配当前页面 DOM 结构
3. 添加 GB2312/GBK 编码处理

### 3.2 fdtz（发改委项目审批）

**失败原因**：加速乐（JSL）CDN Cookie 获取超时，API 请求返回 404 和 500。

**详细诊断**：
- `playwright_utils.wait_for_cookies` 等待 `__jsl_clearance_s`/`__jsl_clearance` Cookie 超时（20 秒内未获取到）
- 即使不带 Cookie 直接请求 API，返回 HTTP 404（路径可能变更）和 HTTP 500（系统异常）
- 加速乐防护策略可能已升级

**修复方向**：
1. 更新 playwright_utils 的 Cookie 等待策略（延长超时、调整等待条件）
2. 验证 API 路径是否变更（`/tzxmspweb/api/publicityInformation/selectByPageBA` 可能不再可用）
3. 可能需要更新加速乐 Challenge 的处理方式
