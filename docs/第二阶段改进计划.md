**技术可行性调研**（详见 `docs/ggzyjy-guangdong-feasibility-report.md`）：

## 三、待实施任务

### 任务 B：其他省份 ggzyjy（江苏/浙江/山东/四川）

**方案**：参照 EIA 多城市适配器模式，为每省建独立适配器。

**执行步骤**：
1. 浏览器 DevTools 抓包验证各省 API 端点
2. 确认可行后在 `scraper/ggzyjy/adapters/` 新增适配器
3. 在 `regions.py` 追加省份配置
4. 不可行的省份果断放弃并记录原因

**预计工作量**：每省 2-3 天

| 省份 | 平台域名 | 可行性 | 说明 |
|------|---------|--------|------|
| 浙江 | ggzy.zj.gov.cn | 部分可行（P1） | 分类体系最完善，URL结构规律性强，需浏览器抓包确认API |
| 江苏 | jsggzy.jszwfw.gov.cn | 部分可行（P2） | 与广东同属“zwfw”政务服务体系，可能有类似API模式 |

### 任务 C：发改委项目审批公示 ✅ 已完成（2026-07-17）

**数据源**：`tzxm.gd.gov.cn`（广东省投资项目在线审批监管平台）

**数据价值**：项目立项/可研批复是最早期的项目情报，比招标早 3-6 个月

**实现概要**：
- 模块结构：`scraper/fdtz/`（__init__.py + api.py + parser.py + utils.py）
- 主类：`FdtzScraper(BaseScraper)`，`source_type = 'fdtz'`
- 采集模式：Playwright 首次加载获取 JSL Cookie → httpx 调用 JSON API（POST 分页+地域筛选）
- 地区覆盖：广东省（tzxm.gd.gov.cn）
- Playwright 基础设施复用：基于 `scraper/playwright_utils.py` 通用浏览器管理

**集成改动**：
- `scraper/registry.py`：追加 fdtz 条目
- `app/config.py`：SCRAPER_SOURCE_SITES 追加 fdtz
- `app/templates/scraper/panel.html`：新增 fdtz 采集卡片

**测试覆盖**：`tests/test_fdtz.py` 单元测试通过

### 发改委项目审批公示调研结论（2026-07-16）

**广东省投资项目在线审批监管平台（tzxm.gd.gov.cn）**：
- 结论：**可行，需 Playwright 突破 JSL 防护**
- API：纯 JSON API（POST），支持分页+地域筛选
- 字段匹配度：高（项目名称、建设单位、投资额、地域、审批日期）
- 建议方案：`scraper/fdtz/` 模块，Playwright 首次加载获取 Cookie 后 httpx 调用 API
- 价值：与 EIA 形成“双前置情报”，比招标早 3-6 个月

**国家发改委（ndrc.gov.cn）**：
- 结论：**不可行**。需登录、Dojo 框架加密、接口返回空数据

> 详细调研过程见 `docs/发改委-采集可行性调研报告.md`
