## 二、已完成工作

### 历史中标方字段提取 ✅

**实现概要**：
- ccgp 中标公告：`extract_winner_info()` 从详情页提取中标供应商名称和中标金额
- gdgpo 成交公告：提升 `winner_name` 到顶层字段，便于列表展示
- 结果存入 `raw_data` JSON 字段，无需 ALTER TABLE 改表结构
- 前端线索详情页展示中标方信息

**技术方案**：
- 中标方信息作为 `raw_data.winner_name` 和 `raw_data.winner_amount` 存储
- ccgp 解析器在 detail.py 中新增中标方提取逻辑
- gdgpo 从 JSON 响应中直接映射 `winnerName` 字段

**测试覆盖**：11 个单元测试，覆盖中标方提取、空值处理、金额格式化等场景

### 项目全生命周期追踪 ✅

**实现概要**：
- `scraper/lifecycle.py`：项目名称标准化 + Jaccard 2-gram 相似度匹配
- 将同一项目的招标、中标、变更、验收等不同阶段公告自动关联
- 匹配结果存入 `raw_data.related_leads`，记录关联线索 ID 列表

**核心算法**：
1. **名称标准化**：去除括号内容、统一全半角、移除常见后缀（招标公告/中标公告/成交公告等）
2. **Jaccard 2-gram 相似度**：将标准化后的名称拆分为 bigram 集合，计算交集/并集比值
3. **阈值匹配**：相似度 ≥ 0.6 视为同一项目

**新增工具函数**（`scraper/utils.py`）：
- `strip_stage_suffix()`：移除公告类型后缀
- `extract_field()`：从 HTML/JSON 安全提取字段
- `tokenized_bigrams()`：生成 token 的 bigram 集合

**测试覆盖**：29 个单元测试，覆盖名称标准化、相似度计算、边界场景等
