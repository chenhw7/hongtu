## 二、EIA 重构经验总结（架构指导原则）

### 2.1 EIA 重构后的目录结构

```
scraper/eia/
├── __init__.py          # EiaScraper 主类（继承 BaseScraper，实现 _scrape_page 分发）
├── regions.py           # 城市注册表（纯常量配置，无逻辑）
├── utils.py             # 纯函数工具集（日期解析、电话提取、KV表格提取、附件提取）
└── adapters/
    ├── __init__.py      # ADAPTERS 注册表 + get_adapter() 工厂函数
    ├── base.py          # BaseAdapter（共享 HTTP 辅助、详情页解析逻辑）
    ├── static.py        # StaticAdapter（约 15 个城市的静态 HTML 采集）
    ├── guangzhou.py     # GuangzhouAdapter（JSON API + 静态 HTML 混合）
    ├── dongguan.py      # DongguanAdapter（POST 表单 API）
    ├── shenzhen.py      # ShenzhenAdapter（.vm 服务端渲染）
    └── zhaoqing.py      # ZhaoqingAdapter（gkmlpt JSON API）
```

### 2.2 核心设计原则

**原则 1：主类薄 — 只做分发和编排**
- `EiaScraper.__init__.py` 只有 80 行，核心逻辑就一个 `_scrape_page()` 方法，根据 `region:xxx` 伪关键词查找适配器并调用
- 不做具体解析，不做 HTTP 细节

**原则 2：适配器独立 — 一城一文件**
- 每个城市/数据源的特殊逻辑封装在独立适配器中
- 适配器之间零耦合，互不引用
- 新增城市只需：① 在 `regions.py` 加配置项 ② 如需特殊逻辑，在 `adapters/` 新增一个文件

**原则 3：公共逻辑上提 — BaseAdapter**
- HTTP 请求辅助（`fetch_html_with_status`、`_post_json`、`_post_form`）
- 详情页通用解析（`parse_detail`）— 提取 KV 表格、项目名称、建设单位、电话等
- 子类只覆盖差异部分

**原则 4：纯函数工具化 — utils.py**
- 日期解析、电话提取、附件提取、公告类型分类等纯函数
- 无外部依赖，无副作用，便于单元测试

**原则 5：注册表模式 — 配置与实现分离**
- `regions.py` 是纯配置（哪个城市用什么 URL、什么选择器、什么适配器）
- `adapters/__init__.py` 的 `ADAPTERS` 字典 + `get_adapter()` 是适配器工厂

### 2.3 本次重构如何应用这些原则

| 场景 | 应用方式 |
|------|---------|
| 新数据源（ctbpsp） | 独立模块 `scraper/ctbpsp/`，内部按功能拆分文件 |
| 现有 ccgp 重构 | 参照 EIA 模式拆分为 `scraper/ccgp/` 模块 |
| 关键词管理 | 独立 `scraper/keywords.py` 纯配置 + 分类，各采集器按需引用 |
| 通用工具 | 将 ccgp/gdgpo 中重复的日期解析、附件提取等纯函数上提到 `scraper/utils.py` |

**关键词子串重叠问题**：

部分关键词存在子串包含关系，同一公告会被多个关键词重复命中，浪费请求次数：
- `管道` ∈ `市政管道`、`供热管网`（语义重叠）
- `波纹管` ∈ `双壁波纹管`
- `给水管` ∈ `PE给水管`
- `排水管` ∈ `排水管网`（语义重叠）

**应对策略**：
- 各采集器的搜索 API 通常返回关键词匹配的标题列表，不会因为子串重叠导致重复入库（`bidding_number` 唯一约束已处理数据级去重）
- 但子串重叠会浪费请求次数：搜索"波纹管"已经覆盖了"双壁波纹管"的结果，再搜一次"双壁波纹管"是多余的
- 因此在组合最终关键词列表时，需去除被其他关键词子串包含的冗余词：

```python
def dedup_substring_keywords(keywords):
    """去除被其他关键词子串包含的冗余词，减少重复搜索"""
    result = []
    for kw in keywords:
        # 如果存在另一个更长的关键词包含此词，则跳过
        if not any(other != kw and kw in other for other in keywords):
            result.append(kw)
    return result

# 最终实际搜索的关键词列表 = 去重后的组合关键词
CCGP_KEYWORDS_FINAL = dedup_substring_keywords(CCGP_KEYWORDS)
EIA_KEYWORDS_FINAL = dedup_substring_keywords(EIA_KEYWORDS)
PLATFORM_KEYWORDS_FINAL = dedup_substring_keywords(PLATFORM_KEYWORDS)
```

**放弃原因**：经真实调研（详见 `docs/ctbpsp-采集可行性调研报告.md`），ctbpsp.com 有 **5 层反爬防护**：
1. 阿里云 WAF（JS 挑战，curl 无法通过）
2. Vaptcha 人机验证（滑块/点击验证码）
3. 网易易盾（备选验证码）
4. 阿里系前端反调试（`interfaceacting.js` + `antidom.js`）
5. DES 响应加密

纯 HTTP 请求方案完全不可行。若要实现需引入 Playwright 浏览器自动化 + 打码平台，成本过高，决定放弃。

#### `ccgp/__init__.py` — 主类（~120 行）
```python
from scraper.keywords import CCGP_KEYWORDS
from scraper.ccgp.regions import REGIONS

class CcgpScraper(BaseScraper):
    source_type = 'ccgp'
    base_url = 'http://search.ccgp.gov.cn/bxsearch'

    def __init__(self, app=None):
        super().__init__(app=app)
        self.ccgp_keywords = CCGP_KEYWORDS

    def default_keywords(self):
        """生成默认关键词列表：关键词搜索词 + 频道伪关键词"""
        keywords = list(self.ccgp_keywords)
        keywords.append('channel:zygg')  # 中央公告频道
        # 31 个省级行政区的地方公告频道
        for region_code in REGIONS:
            keywords.append(f'channel:dfgg:{region_code}')
        return keywords

    def _scrape_page(self, keyword, page):
        # 分发到 search.py 或 channel.py
        if keyword.startswith('channel:'):
            return channel.scrape_channel_page(self, keyword, page)
        return search.scrape_search_page(self, keyword, page)

    def run(self, keywords=None, max_pages=None):
        """
        关键词搜索默认 max_pages=5，频道采集默认 max_pages=2。
        分省采集的单省公告量有限，无需翻太多页。
        """
        if keywords is None:
            keywords = self.default_keywords()
        # 按关键词类型分配不同的 max_pages
        if max_pages is None:
            results = []
            for kw in keywords:
                kw_max_pages = 2 if kw.startswith('channel:') else 5
                results.extend(super().run(keywords=[kw], max_pages=kw_max_pages))
            return results
        # 手动指定 keywords 时，统一使用传入的 max_pages
        return super().run(keywords=keywords, max_pages=max_pages)
```

## 五、风险与注意事项（实际执行回顾）

### 5.1 ccgp 重构风险
- **风险**：重构可能破坏现有的 ccgp 采集功能
- **实际处理**：
  - 重构前保留了 `scraper/ccgp.py.bak` 备份
  - 模块化后 `from scraper.ccgp import CcgpScraper` 路径不变（通过 `__init__.py` 导出）
  - 重构与原采集功能兼容

### 5.2 ctbpsp 接口不确定性（已确认）
- **调研结论**：ctbpsp 有 5 层反爬保护（WAF + 验证码 + 反调试 + 加密），纯 HTTP 请求不可行
- **决策**：放弃，详见 `docs/ctbpsp-采集可行性调研报告.md`

### 5.3 分省采集的请求量增加
- **风险**：从 2 个频道扩展到 32 个频道（1 中央 + 31 省）+ 约 40 个搜索关键词 = 72 个采集单元，请求量大幅增加
- **缓解**：频道采集 `max_pages` 设为 2（而非搜索关键词的 5），通过 `CcgpScraper.run()` 按关键词类型分配 max_pages；`scheduler.py` 中已错开各采集器执行时间

### 5.4 速率控制与反爬风险
- **风险**：72 个采集单元 × 每单元 2~5 页 = 约 200~360 次请求，单 IP 高频请求可能被 ccgp/ctbpsp 封禁
- **缓解**：
  - `BaseScraper` 已有请求间隔机制，重构后确认间隔时间不低于 1 秒
  - 各采集器串行执行（不在同一时间并发多个采集单元），通过 `scheduler.py` 错开时间
  - 如被封禁，在 `scheduler.py` 中增加更长的间隔（如 3~5 秒），或按省份分批执行（每批 5 省，间隔 10 分钟）

### 5.5 数据库去重
- **风险**：分省采集可能导致同一公告被多次采集（某公告同时出现在省频道和全国频道）
- **缓解**：现有的 `bidding_number` 唯一约束已经处理了去重，`save_leads()` 在遇到重复时会跳过
