# 其他省份公共资源交易平台（ggzyjy）采集可行性调研报告

> 调研日期：2026-07-16
> 调研范围：浙江、江苏、四川、山东四省公共资源交易平台
> 状态：**调研完成，待浏览器抓包验证**
> 参考：广东省 ggzyjy 已成功接入（详见 `ggzyjy-guangdong-feasibility-report.md`）

---

## 一、调研背景

广东省公共资源交易平台（粤公平）已成功接入，验证了 ggzyjy 类平台的技术可行性（REST API 直连、无反爬、结构化 JSON 返回）。现对其他四省同类平台进行可行性调研，评估扩展至全国多省的可行性。

### 调研方法

- 平台官网访问，分析前端架构和 URL 结构
- 对比广东省 ggzyjy 的技术模式（Vue.js SPA + REST API）
- 评估 API 可直连性和反爬强度

---

## 二、浙江省公共资源交易平台

### 基本信息

| 项目 | 信息 |
|------|------|
| 平台名称 | 浙江省公共资源交易平台 |
| 官方域名 | ggzy.zj.gov.cn |
| 覆盖范围 | 全省 11 个地级市 + 省级 |
| 主要板块 | 工程建设、政府采购、土地交易、产权交易 |

### 技术架构

- **前端框架**：Vue.js SPA，与广东省类似
- **URL 结构**：规律性强，分类体系最完善
- **API 模式**：推测与广东省类似的 REST API 架构

### API 可直连性

- 预期存在搜索/列表/详情三类 API 端点
- URL 结构规范，便于批量采集
- **待验证**：需浏览器 DevTools 抓包确认具体 API 端点和参数格式

### 反爬强度

- **评估**：低。省级政府公共服务平台，信息公开属性强
- **风险**：可能存在基础频率限制
- **结论**：预期与广东省类似，httpx 直连可行

### 可行性结论

**部分可行（优先级 P1）**。分类体系最完善，URL 结构规律性强，技术架构与广东省相似度高，最有可能复用现有 `scraper/ggzyjy/` 模块架构。需浏览器抓包确认 API 后即可开发。

## 三、江苏省公共资源交易平台

### 基本信息

| 项目 | 信息 |
|------|------|
| 平台名称 | 江苏省公共资源交易平台 |
| 官方域名 | jsggzy.jszwfw.gov.cn |
| 覆盖范围 | 全省 13 个地级市 + 省级 |
| 主要板块 | 工程建设、政府采购、水利工程、交通工程 |

### 技术架构

- **前端框架**：SPA 架构
- **URL 结构**：与广东同属"zwfw"（政务服务）体系
- **API 模式**：域名中包含 `zwfw`，与广东省 `gdzwfw.gov.cn` 同体系，可能有类似 API 模式

### API 可直连性

- "zwfw" 政务服务体系下，API 设计模式可能高度一致
- 推测搜索/分页/详情接口与广东省有相似结构
- **待验证**：需浏览器 DevTools 抓包确认

### 反爬强度

- **评估**：低。同属政务服务体系统一建设标准
- **风险**：可能存在 IP 频率限制
- **结论**：预期 httpx 直连可行

### 可行性结论

**部分可行（优先级 P2）**。与广东同属"zwfw"政务服务体系，API 模式可能高度相似，开发成本预期较低。需浏览器抓包验证后确定具体接口差异。

## 四、四川省公共资源交易平台

### 基本信息

| 项目 | 信息 |
|------|------|
| 平台名称 | 四川省公共资源交易平台 |
| 官方域名 | ggzyjy.sc.gov.cn |
| 覆盖范围 | 全省 21 个市州 + 省级 |
| 主要板块 | 工程建设、政府采购、国有产权、土地矿权 |

### 技术架构

- **前端框架**：SPA 架构
- **URL 结构**：部分页面使用 8443 非标端口
- **API 模式**：存在端口不一致的情况，增加适配复杂度

### API 可直连性

- 部分 URL 使用 `https://ggzyjy.sc.gov.cn:8443/` 端口
- API 端点格式待确认，可能与广东省有较大差异
- **待验证**：需浏览器 DevTools 抓包确认端口策略和 API 端点

### 反爬强度

- **评估**：低-中。政府公开信息，但非标端口可能暗示部署架构较旧
- **风险**：8443 端口可能有 SSL 证书问题，httpx 需特殊处理
- **结论**：技术上可行，但需额外适配工作

### 可行性结论

**部分可行（优先级 P3）**。覆盖 21 市州（与广东省地市数量一致），数据价值高。但 8443 非标端口增加了适配复杂度，建议放在浙江/江苏之后实施。

## 五、山东省公共资源交易平台

### 基本信息

| 项目 | 信息 |
|------|------|
| 平台名称 | 山东省公共资源交易平台 |
| 官方域名 | ggzyjy.shandong.gov.cn |
| 覆盖范围 | 全省 16 个地级市 + 省级 |
| 主要板块 | 工程建设、政府采购、土地交易 |

### 技术架构

- **平台状态**：旧平台返回 502 错误，疑似正在迁移
- **新系统**：需要登录认证才能查询公告信息
- **API 模式**：无法确认，旧系统已不可用

### API 可直连性

- 旧平台 502 不可访问
- 新系统需登录，无法匿名获取数据
- **结论**：当前无法接入

### 反爬强度

- **评估**：不适用（平台不可用）
- **风险**：新系统登录认证构成根本性障碍
- **结论**：不可行

### 可行性结论

**不可行（优先级 P4）**。旧平台已下线（502），新系统要求登录认证，暂放弃。待平台迁移完成且开放匿名查询后重新评估。
