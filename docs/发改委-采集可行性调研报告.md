# 发改委项目审批公示 — 采集可行性调研报告

> 调研日期：2026-07-16
> 状态：**调研完成，待 Playwright 方案验证**
> 调研范围：广东省投资项目在线审批监管平台、国家发改委

---

## 一、调研背景

发改委项目审批公示是项目生命周期中最早期的情报节点（立项/可研批复/初步设计审批），比招标早 3-6 个月。如果能采集到此类数据，将与现有 EIA（环评公示）形成"双前置情报"体系，大幅提升销售线索的先发优势。

### 调研目标

- 评估广东省发改委平台（tzxm.gd.gov.cn）的采集可行性
- 评估国家发改委（ndrc.gov.cn）的采集可行性
- 确定 API 端点、请求参数、字段映射、反爬强度
- 给出实现方案建议

---

## 二、广东省投资项目在线审批监管平台

### 2.1 平台概览

| 项目 | 信息 |
|------|------|
| 平台名称 | 广东省投资项目在线审批监管平台 |
| 官方域名 | tzxm.gd.gov.cn |
| 主管单位 | 广东省发展和改革委员会 |
| 数据类型 | 投资项目立项/备案/核准/审批公示 |
| 覆盖范围 | 广东省各地市 |

### 2.2 API 端点分析

#### 搜索/列表接口

```
POST https://tzxm.gd.gov.cn/[项目审批公示列表接口]
Content-Type: application/json
```

**请求参数**（JSON Body，推测）：

| 参数 | 类型 | 说明 |
|------|------|------|
| pageNo | int | 页码 |
| pageSize | int | 每页条数 |
| regionCode | string | 地域编码（市/区县） |
| projectType | string | 项目类型 |
| startTime | string | 起始时间 |
| endTime | string | 结束时间 |

**响应格式**：纯 JSON API（POST），支持分页 + 地域筛选

#### 详情接口

详情页提供项目审批的完整信息，包括项目名称、建设单位、投资额、审批日期等。

### 2.3 字段映射表

| Lead 模型字段 | 来源字段 | 可获取 | 说明 |
|--------------|---------|--------|------|
| project_name | 项目名称 | ✅ | 投资项目审批公示中的项目名称 |
| purchaser | 建设单位 | ✅ | 项目建设单位/投资方 |
| budget_amount | 投资额/总投资 | ✅ | 项目总投资额 |
| region | 所属地域 | ✅ | 项目所在地市/区县 |
| publish_date | 审批日期/公示日期 | ✅ | 审批结果公示时间 |
| notice_type | 审批类型 | ✅ | 立项/备案/核准/审批 |
| bidding_number | 项目代码 | ⚠️ | 投资项目统一代码（非招标编号） |
| contact_person | — | ❌ | 审批公示通常不含联系人 |
| contact_phone | — | ❌ | 审批公示通常不含联系电话 |
| attachments | 批复文件 | ⚠️ | 部分公示可能附带批复文件 |

**匹配度评估**：**高**。虽然没有联系人/电话（招标阶段才有），但项目名称、建设单位、投资额、地域等核心字段均可获取，足以作为前置情报线索。

### 2.4 反爬评估

| 防护维度 | 情况 | 说明 |
|---------|------|------|
| WAF | JSL（极验）防护 | 首次访问需通过 JSL 验证获取 Cookie |
| 验证码 | JSL 滑块/点击验证 | 需 Playwright 浏览器自动化通过 |
| Cookie/Token | 需 JSL Cookie | 通过后 httpx 可携带 Cookie 继续请求 |
| 登录要求 | 不需要 | 公示信息无需登录 |
| 响应加密 | 无 | API 返回明文 JSON |
| 频率限制 | 未确认 | 预期存在基础频率限制 |

**反爬结论：中等（JSL 防护）**。比广东省 ggzyjy（0层防护）严格，但远不如 ctbpsp（5层反爬）。JSL 防护可通过 Playwright 首次加载突破。

### 2.5 实现方案建议

**推荐方案：Playwright + httpx 混合模式**

```
┌──────────────────────────────────────────────────┐
│  Playwright（一次性）                              │
│  1. 打开 tzxm.gd.gov.cn                          │
│  2. 通过 JSL 验证                                 │
│  3. 提取 Cookie                                   │
│  4. 关闭浏览器                                     │
└──────────────┬───────────────────────────────────┘
               │ Cookie
               ▼
┌──────────────────────────────────────────────────┐
│  httpx（批量采集）                                  │
│  1. 携带 Cookie 调用 JSON API                      │
│  2. 分页遍历 + 地域筛选                             │
│  3. 解析 JSON 入库                                 │
│  4. Cookie 过期后重新执行 Playwright 步骤           │
└──────────────────────────────────────────────────┘
```

**模块结构建议**：

```
scraper/fdtz/
├── __init__.py    # FdtzScraper 主类（继承 BaseScraper）
├── regions.py     # 广东省各地市编码配置
├── search.py      # 列表 API + 分页逻辑
├── detail.py      # 详情解析
├── parser.py      # 字段映射
└── utils.py       # Cookie 管理 + JSL 辅助
```

**source_type**：`fdtz`（发改投资）

## 三、国家发改委

### 3.1 平台概览

| 项目 | 信息 |
|------|------|
| 平台名称 | 国家发展和改革委员会 |
| 官方域名 | ndrc.gov.cn |
| 数据类型 | 全国投资项目审批/核准/备案 |

### 3.2 可行性评估

| 评估维度 | 情况 | 说明 |
|---------|------|------|
| 登录要求 | ❌ 需登录 | 查询接口需要用户登录 |
| 前端框架 | Dojo 框架 | JavaScript 加密混淆严重 |
| 接口返回 | 空数据 | 接口调用后返回空数据，无法正常获取 |
| 反爬强度 | 高 | 多层防护机制 |

### 3.3 可行性结论

**不可行。** 具体原因：

1. **需登录**：查询接口不对外公开，需要账号登录
2. **Dojo 框架加密**：前端 JavaScript 采用 Dojo 框架，代码加密混淆严重，难以逆向
3. **接口返回空数据**：即使绕过前端，API 返回空数据，无法正常获取信息
4. **投入产出比低**：广东省平台已覆盖本省需求，国家发改委的额外增量有限

## 四、结论与建议

### 总体结论

| 平台 | 结论 | 原因 |
|------|------|------|
| 广东省 tzxm.gd.gov.cn | **可行** | JSON API + Playwright 可突破 JSL 防护 |
| 国家发改委 ndrc.gov.cn | **不可行** | 登录 + Dojo 加密 + 空数据 |

### 建议

1. **优先开发广东省平台**（`scraper/fdtz/` 模块），与 EIA 形成双前置情报
2. **暂缓国家发改委**，关注其后续是否开放公开查询接口
3. **技术验证**：先用 Playwright 手动验证 JSL 突破方案，确认 Cookie 有效期
4. **开发周期**：预计 2-3 天（含 Playwright 集成 + API 对接 + 测试）

### 风险与缓解

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|---------|
| JSL Cookie 频繁过期 | 中 | 需频繁重新获取 | 监控 Cookie 有效期，自动刷新 |
| Playwright 部署复杂度 | 中 | 增加环境依赖 | 容器化部署或独立进程 |
| API 端点变更 | 低 | 采集中断 | 模块化设计便于修复 |
| 审批公示信息不全 | 低 | 线索质量下降 | 与 EIA 数据交叉验证 |

---

> **决策建议**：投入 2-3 天开发广东省发改委平台采集模块，实现"双前置情报"体系。国家发改委暂放弃。
