## 一、当前采集状态分析

### 1.1 是否已有采集器？

**否。** 当前项目 `scraper/registry.py` 中未注册 `qyggzy` 数据源，代码库中无任何对 `qyggzy.cn` 的引用。

### 1.2 是否有替代覆盖？

**有。** 现有 `ggzyjy` 采集器（广东省公共资源交易平台/粤公平）已覆盖清远市数据：

| 对比项 | ggzyjy（粤公平） | qyggzy（清远市平台） |
|--------|-----------------|---------------------|
| 域名 | ygp.gdzwfw.gov.cn | www.qyggzy.cn:8080 |
| 层级 | 省级聚合平台 | 市级独立平台 |
| 清远编码 | siteCode: 441800 | xzbm: 4418 |
| 采集方式 | POST JSON API | JSP 服务端渲染 |
| 当前状态 | ✅ 已实现，含清远地区 | ❌ 未采集 |

**关键判断**：粤公平平台作为省级聚合平台，理论上汇聚了清远市平台的数据。当前 ggzyjy 采集器通过关键词搜索（全省范围）已能捕获清远市的工程建设招标信息。

### 1.3 两者数据差异

通过对比发现：
- 清远市平台的招标编号格式为 `GC4418002026146`（市级编号体系）
- 粤公平平台的编号格式为 `E4401002701503104001`（省级编号体系）
- 两个平台**可能存在数据同步延迟**，市级平台通常更新更快
- 市级平台可能包含部分**未同步到省级的区县级小项目**

## 三、采集价值评估

### 3.1 数据价值

| 维度 | 评分 | 说明 |
|------|:----:|------|
| 项目相关性 | ⭐⭐⭐⭐ | 清远市政/基建项目直接关联管道建材需求 |
| 数据时效性 | ⭐⭐⭐⭐ | 市级平台更新快，招标→开标周期通常 20-30 天 |
| 字段完整度 | ⭐⭐⭐ | 列表页有项目名/金额/单位，但缺联系方式 |
| 数据规模 | ⭐⭐ | 清远为三四线城市，日均新增约 5-15 条工程建设公告 |
| 增量价值 | ⭐⭐ | 与 ggzyjy（粤公平）存在较大重叠 |

### 3.2 与现有数据源的重叠分析

**高度重叠**：ggzyjy 采集器已覆盖清远市（siteCode 441800）的工程建设招标数据。

实际验证：
- ggzyjy 的关键词搜索（如"管道"、"给排水"、"市政管网"）覆盖全省，包含清远
- ggzyjy 的频道模式（channel:jsgc）浏览全省工程建设，也包含清远数据
- 粤公平平台的设计目标就是汇聚各地市交易数据

**可能的增量**：
- 区县级小型项目（如阳山镇级工程）可能未及时同步到省级平台
- 市级平台更新可能比省级快 1-2 天
- 预计增量占比：10-20%（大部分数据省级平台已有）

### 3.3 结论

**采集价值有限，投入产出比不高。**

理由：
1. ggzyjy 已覆盖清远市 80-90% 的招标数据
2. 清远为三四线城市，项目总量有限
3. 核心联系方式在 PDF 中，仅靠列表页无法获得完整线索
4. 单独开发采集器 + PDF 解析的成本较高

## 四、采集难点分析

### 4.1 PDF 内容解析（核心难点）

| 难点 | 说明 |
|------|------|
| PDF 格式不统一 | 不同代理机构出具的招标文件格式各异 |
| 文件体积大 | 单个招标文件通常 5-50MB，含图纸 |
| 解析成本高 | 需下载→OCR/文本提取→结构化，每条耗时数秒至数十秒 |
| 信息位置不固定 | 联系人/电话可能在任意页面 |
| 存储压力 | 日积月累 PDF 文件占用大量磁盘空间 |

### 4.2 技术层面

| 难点 | 说明 |
|------|------|
| JSP 分页机制 | 需分析表单提交或 URL 参数分页逻辑 |
| 编码问题 | 老系统可能使用 GBK 编码 |
| HTTPS 证书 | 8080 端口 + 自签名证书（需忽略验证） |
| 详情页 URL 规则 | 需通过浏览器抓包确认实际链接格式 |
| 无公开 API | 纯 HTML 页面，无 JSON API 可用 |

### 4.3 反爬评估

| 防护维度 | 评估 |
|---------|------|
| WAF | 无（政府公开平台） |
| 验证码 | 无 |
| 登录要求 | 无（列表浏览无需登录） |
| 频率限制 | 可能有基础限制 |
| PDF 下载限制 | 可能需要登录或验证码（待验证） |

**反爬强度：极低**（列表页），PDF 下载可能有额外限制。

## 五、采集方案（如需实施）

### 方案 A：仅采集列表页（推荐，若确需实施）

**思路**：放弃 PDF 解析，仅从列表页提取结构化数据。

```
技术栈：httpx + BeautifulSoup/lxml
工作量：1.5-2 天
```

**实现步骤**：
1. 分析列表页 HTML 结构，确定分页参数
2. 新建 `scraper/qyggzy/` 模块
3. 解析列表项字段（项目名、编号、金额、单位等）
4. 注册到 `SCRAPER_REGISTRY`
5. 复用 BaseScraper 的去重/限速/进度机制

**优点**：实现简单，无需处理 PDF
**缺点**：缺少联系方式，线索完整度不足

### 方案 B：列表页 + PDF 解析（完整方案）

**思路**：下载 PDF 招标文件，提取联系人和材料需求信息。

```
技术栈：httpx + BeautifulSoup + PyMuPDF(pdfplumber) + 正则
工作量：4-6 天
```

**实现步骤**：
1. 列表页采集（同方案 A）
2. 获取 PDF 下载链接
3. 下载 PDF 到 `instance/attachments/qyggzy/`
4. 使用 pdfplumber 提取文本
5. 正则匹配联系人/电话/材料清单
6. 结构化入库

**PDF 解析策略**：
```python
import pdfplumber

def extract_contacts_from_pdf(pdf_path):
    """从招标文件 PDF 中提取联系方式。"""
    contacts = {}
    with pdfplumber.open(pdf_path) as pdf:
        # 通常联系信息在前 5 页或"投标人须知"章节
        for page in pdf.pages[:10]:
            text = page.extract_text() or ''
            # 匹配电话
            phones = re.findall(r'(?:电话|联系电话|Tel)[：:]\s*([\d\-]+)', text)
            # 匹配联系人
            persons = re.findall(r'(?:联系人|联 系 人)[：:]\s*(\S{2,4})', text)
            if phones:
                contacts['phone'] = phones[0]
            if persons:
                contacts['contact_person'] = persons[0]
    return contacts
```

**优点**：线索完整度高，可获取联系方式
**缺点**：开发成本高、PDF 格式不统一导致解析不稳定、存储开销大

### 方案 C：增强现有 ggzyjy 采集器（最优方案）

**思路**：不新建采集器，而是优化现有 ggzyjy 对清远地区的覆盖。

```
工作量：0.5 天
```

**实现步骤**：
1. 在 ggzyjy 的 `default_keywords()` 中增加清远地区频道：
   ```python
   keywords.append('channel:jsgc:qingyuan')  # 清远市工程建设
   ```
2. 验证 ggzyjy 对清远数据的覆盖率
3. 如有遗漏，调整采集频率或增加关键词

**优点**：零新增代码，复用现有成熟架构
**缺点**：依赖省级平台数据同步，可能有 1-2 天延迟

## 六、最终建议

### 推荐方案：方案 C（增强 ggzyjy）+ 观察

| 优先级 | 行动 | 投入 |
|:------:|------|------|
| P0 | 在 ggzyjy 中启用 `channel:jsgc:qingyuan` 频道采集 | 0.5 天 |
| P1 | 运行 1-2 周，对比 ggzyjy 与 qyggzy 的数据覆盖率 | 观察 |
| P2 | 如确认存在显著数据缺口（>20%），再考虑方案 A | 视情况 |
| P3 | PDF 解析仅在业务强需求时考虑（方案 B） | 暂不推荐 |

### 不建议立即投入的理由

1. **数据重叠率高**：粤公平平台已覆盖清远市大部分招标数据
2. **增量有限**：清远为三四线城市，日均新增项目少
3. **PDF 解析 ROI 低**：高开发成本 + 不稳定的解析效果
4. **现有方案可补位**：ggzyjy 增加清远频道即可覆盖

### 未来可重新评估的时机

- 业务重点拓展清远市场时
- 发现 ggzyjy 对清远数据覆盖率明显不足时
- PDF 解析技术成本降低时（如接入 LLM 文档解析）
